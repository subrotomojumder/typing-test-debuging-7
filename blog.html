Q.1. What is the difference between Local Storage and Session Storage?
Ans:

Local Storage:- local storage URL বা উইন্ডোতে টিকে থাকে । user site close করলেও, user এর experience local storage থেকে যায় । পরে ‍কখনো আগের ঐ site browsing করলে page url / user interface আগের মতো দেখায় । local storage এ সংরক্ষিত data মেয়াদ শেষ হওয়ার তারিখ নেই এবং শুধুমাত্র Java script এর মাধ্যমে browser Local storage এ সংরক্ষিত data clear করা হয়। local storage ‍সংরক্ষন করার ability session storage এ থেকে অনেক বেশি । local storage এ সংরক্ষিত data স্পষ্টভাবে মুছে না যাওয়া পর্যন্ত টিকে থাকে। করা পরিবর্তনগুলি সংরক্ষিত এবং সাইটের বর্তমান এবং ভবিষ্যতের সমস্ত পরিদর্শনের জন্য উপলব্ধ। user এর গোপনীয় নয় এমন কিছু ডেটা ধরে রাখতে, local storage ব্যবহার করতে হয় ।

Session Storage:- session storage local storage এর অনুরূপ । একবার window বন্ধ হয়ে গেলে, session storage মুছে ফেলা হয় । session storage ডেটা শুধুমাত্র সেই window এর tab এ পাওয়া যায় যে tab এ browsing করা হয়েছিল । Session Storage ডেটা স্থায়ী নয় অর্থাৎ উইন্ডো/ট্যাব বন্ধ হয়ে গেলে এটি হারিয়ে যাবে। local storage এর মতো, এটি একই origin policy তে কাজ করে। সুতরাং, সংরক্ষিত ডেটা একই মাধ্যমে পাওয়া যাবে। session storage property প্রতিটি প্রদত্ত মূলের জন্য একটি পৃথক স্টোরেজ এলাকা বজায় রাখে যা page session সময়কালের জন্য বরাদ্দ থাকে, যতক্ষণ পর্যন্ত browser খোলা থাকে, পৃষ্ঠা পুনরায় লোড করা এবং পুনরুদ্ধার করার কাজ session storage করে থাকে।

Q.2. difference between Global scope and Block scope.
Ans:

Block Scope:- Block Scope সাধারনত curly bracket ({----}) এর ভিতরের অংশকে বুঝায় । Java Script ES6 (2015) এর আগে, Java Script এ শুধুমাত্র Global scope এবং ফাংশন স্কোপ ছিল। ES6 দুটি গুরুত্বপূর্ণ নতুন Java Script special keyword প্রবর্তন করেছে: let এবং const। এই দুটি keyword Java script Block Scope প্রদান করে । একটি { } ব্লকের ভিতরে ঘোষিত ভেরিয়েবলগুলি ব্লকের বাইরে থেকে অ্যাক্সেস করা যাবে না ।

Global Scope:- একটি programming এনভায়রনমেন্টে, global scope হল সেই scope যা অন্যান্য সমস্ত স্কোপের মধ্যে রয়েছে এবং দৃশ্যমান। একটি Java script program এ যেকোনো জায়গা থেকে global variable access করা যেতে পারে। var, let এবং const দিয়ে ঘোষিত ভেরিয়েবলগুলি যখন ব্লকের বাইরে ঘোষণা করা হয় তখন variable global হয়ে থাকে । HTML-এ, Global scope হল windhow object।

Q.3. How Java Script Event Loop works?

Ans: Java script একটি single threaded synchronous programming language । Java script asynchronous ‍and synchronous code গুলিকে run করে event loop system এর মাধ্যমে । Event loop এর একটা অংশ হলো Browser Engine যা programming language কে machine langauge এ convert করে । Browser Engine এর দুটি অংশ 1.Heap 2.Call stack । call stack Java script এর code গুলি axecuted করে । call stack প্রথমে একে একে সব code কে নেয় এবং synchronous গুলোকে run করে এবং asynchronous গুলোকে Queue তে রাখে । callstack synchronous code run complete করে event loop করে Queue থেকে asynchronous code run complete করে ।

Q.4. How many ways to get undefiend?
    Ans:
  1. variable declare করে value না দিয়ে ঐ variable কে access করতে চাইলে ।
  2. Function এর কোন arguments pass করতে mistake করলে arguments return undefiend আসবে ।
  3. কোন Function return লিখে return value না দিলে
  4. অবজেক্টের property access করলে ওই object এ যদি ওই property না থাকে undefiend আসবে ।
  5. array এর কোন index value access করলে ওই index এর value না থাকলে undefiend আসবে ।
